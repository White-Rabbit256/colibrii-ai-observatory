# Contribution Guidelines

Thank you for your interest in contributing to **Awesome AI Governance LATAM**! This list aims to be the most comprehensive collection of AI governance, policy intelligence, and readiness resources for Latin America and emerging markets.

## How to Contribute

### Adding a Resource

1. **Fork** this repository.
2. **Create a branch** with a descriptive name: `add-resource-name`.
3. **Add your resource** to the appropriate category in `README.md`.
4. **Submit a pull request** using the provided template.

### Resource Format

Each entry must follow this format:

```markdown
- [Resource Name](https://example.com/) - Brief description in one sentence. `[Tag1]` `[Tag2]`
```

**Rules:**
- The description must be concise (one sentence, no more than 150 characters).
- The description must start with a capital letter and end with a period.
- The link must point to a live, accessible URL.
- Use the appropriate tags (see below).

### Available Tags

| Tag | Meaning |
|-----|---------|
| `[Free]` | Resource is freely accessible |
| `[Paid]` | Requires payment or subscription |
| `[Free Tier]` | Has a free tier with paid options |
| `[API]` | Provides a programmatic API |
| `[Open Source]` | Source code is publicly available |
| `[Spanish]` | Available in Spanish |
| `[Portuguese]` | Available in Portuguese |
| `[Costa Rica]` | Specific to Costa Rica |

### Quality Standards

A resource will be accepted if it meets these criteria:

- **Relevant**: Directly related to AI governance, policy, readiness, or ethics with relevance to Latin America or emerging markets.
- **Maintained**: The resource must be actively maintained (updated within the last 2 years).
- **Accessible**: The URL must be live and accessible.
- **Non-duplicate**: The resource must not already be listed.
- **Non-promotional**: Resources should provide genuine value. Commercial products are acceptable if they offer significant governance utility.

### Categories

If your resource does not fit an existing category, you may propose a new one. New categories require at least 3 initial resources.

### What We Do Not Accept

- Resources that are primarily marketing material with no governance substance.
- Dead links or resources behind paywalls with no free alternative.
- Resources exclusively in languages other than English, Spanish, or Portuguese (exceptions considered case by case).
- Duplicate entries or minor variations of existing resources.

## Updating Existing Resources

If a resource URL has changed, description needs updating, or it should be recategorized:

1. Open an issue describing the change needed, OR
2. Submit a pull request with the correction.

## Reporting Issues

- **Dead links**: Open an issue with the title `Dead link: [Resource Name]`.
- **Incorrect information**: Open an issue with the title `Correction: [Resource Name]`.
- **Recategorization**: Open an issue explaining why a resource belongs in a different category.
